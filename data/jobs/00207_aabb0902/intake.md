Job Summary and Responsibilities

Job Summary / Purpose
The SSRM Data Analytics Engineer serves as the intersection of the supply chain business aspects, data analytics, and data engineering, and is responsible for bringing robust, efficient, and integrated data models and reporting to life. This role will integrate and prepare varied and disparate datasets by using Data Build Tool (dbt for documentation and code automation), Version Control Systems (such as GitHub), Integrated Development Environment (IDE such as Visual Studio) and Python-based code (i.e. Prefect data flows) to organize and document data tables.

Essential Key Job Responsibilities
Provides clean data sets to end users, modeling data in a way that empowers end users to answer pertinent supply chain questions.
Extends knowledge of supply chain procurement and inventory best practices to transform raw ERP (Enterprise Resource Planning) data into consumable, actionable information which leads to item master, par, and procurement optimization.
Provides support to other SSRM analyst roles for advancing code and datasets within the department
Partners with data engineers to build improved data infrastructure
Normalizes and integrates new procurement transaction data into existing datasets as CommonSpirit Health grows through acquisitions
Ensures visual analytics are optimized within CommonSpirit Health's standard tools for supporting reporting and analytics
Uses or develops skills to use python-based tools to automate reports/data per end user requirements.
