Full job description
InnoVet Health, a small and growing business that provides health IT professional services to the Department of Veterans Affairs (VA) is looking for an experienced Dev Ops Data Engineer. You will design, implement, and own CI/CD pipelines for ADF, Databricks (notebooks/jobs/libraries), and Power BI using GitHub Actions. Your work will directly impact VA healthcare delivery by building an analytical infrastructure that enables AI/ML and advanced analytics to deliver short-term and longitudinal insights to healthcare professionals serving Veterans. You will work in an agile environment alongside VA and contractor stakeholders. It is flexible, full-time, and does not require relocation (work-from-home). The pay, benefits, and growth potential are competitive.

Responsibilities

CI/CD & Release Management

Design, implement, and own CI/CD pipelines for ADF, Databricks (notebooks/jobs/libraries), and Power BI using GitHub Actions (or equivalent).
Manage and optimize Power BI Deployment Pipelines across Dev/Test/Prod with parameterization and artifact promotion.
Define and enforce branching strategies, PR reviews, release gates, and semantic versioning across repos.
Introduce blue/green/canary strategies for critical pipelines, enabling rapid rollback and drift detection.
Maintain build artifact registries (e.g., wheels/packages for Databricks, shared SQL scripts), enforcing immutable builds.
Infrastructure & Cloud Operations

· Manage and optimize Azure infrastructure across ADLS, ADF, Databricks, Key Vault, networking, and access boundaries.

· Optimize orchestrations, cluster sizing, job configs, and SQL Warehousing for throughput and cost efficiency.

· Monitor platform health, performance, and costs; proactively resolve bottlenecks and failures.

Security & Compliance (Healthcare)

Enforce least-privilege access, RBAC, Managed Identities, and secure networking (e.g., Private Endpoints).
Manage secrets/credentials via Azure Key Vault; ensure sensitive data is never exposed in source control.
Partner with InfoSec & Compliance to implement policy-as-code, governance baselines, and periodic controls testing.
Reliability, Observability & SRE

Build end-to-end monitoring and alerting for ADF triggers & activity runs, Databricks jobs, SQL jobs, and Power BI refresh using Azure Monitor & Log Analytics.
Define and track SLIs/SLOs (e.g., data freshness, pipeline success rates, report refresh reliability).
Lead incident response, postmortems, and continuous improvement cycles; publish dashboards and runbooks.
Testing & Release Quality

Implement automated testing for data (schema checks, constraints, data quality rules), notebooks, SQL objects, and Power BI artifacts.
Integrate data quality frameworks (Great Expectations, dbt tests) into CI/CD with clear pass/fail gates.
Standardize test environments, code coverage, integration tests, and reproducible builds.
Governance & FinOps

Enforce naming standards, tagging, resource lifecycle policies, and tenant/workspace governance (incl. Power BI capacity planning).
Monitor and manage compute/storage costs (Databricks clusters, ADLS tiers, Power BI capacity); establish budgets and guardrails.
Collaboration, Enablement & Leadership
Serve as technical coach for data engineers and BI developers on DevOps best practices.
Partner cross-functionally with Analytics, Security, and Infrastructure teams to standardize environments and deployment practices.
Act as the primary escalation point for DevOps-related issues impacting pipelines or BI refreshes.
Create and maintain documentation (runbooks, architecture diagrams, operational procedures) and run workshops to improve self-service and developer velocity.
Qualifications

6+ years in DevOps/Platform/Cloud/Data Engineering, with 2–3+ years leading DevOps initiatives for data teams.
Hands-on CI/CD experience for:
Azure Data Factory (ADF JSON, adf_publish,)
Azure Databricks (Workspace/Repos, Jobs, Clusters, Libraries; deployments via GitHub Actions)
Power BI (Deployment Pipelines, artifact promotion, parameterization, dataset refresh orchestration)
Strong with Git/GitHub (Actions, environments, runners) and release management.
Proficiency in SQL/T-SQL, data modeling basics, and ETL/ELT pipeline patterns.
Solid grasp of Azure security: Key Vault, RBAC, Managed Identities, Private Endpoints, networking fundamentals.
Proven experience with monitoring/observability (Azure Monitor, Log Analytics; Application Insights helpful).
Excellent communication, documentation, and stakeholder management skills.
Preferred

Experience with Databricks SQL Warehousing, Delta Lake, and Unity Catalog.
Data quality frameworks and test automation (e.g., Great Expectations, dbt tests).
Familiarity with Power BI service administration, capacity planning, and tenant governance.
Prior healthcare experience; knowledge of HIPAA, HITRUST, or equivalent compliance frameworks.
Familiarity with VA EHR systems (Oracle Health, Vista/CPRS, CDW etc)
Microsoft certifications: AZ-400 (DevOps Engineer Expert), AZ-104 (Azure Admin), DP-203 (Data Engineer).
Exposure to Azure DevOps pipelines/boards used alongside GitHub.
Experience with FinOps and Azure Policy (policy-as-code).
Familiarity with VA EHR systems like (Oracle, CDW, Vista/CPRS etc.) This job isn't just coding so being familiar with healthcare data is extremely helpful and the nuances with working with VA data in general.
