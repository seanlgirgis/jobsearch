* **0** for jobs rejected by Tier 0 local gate

## REQUIRED ARCHITECTURE

## Tier 0 — Local Gate

Tier 0 runs before any paid LLM and must produce:

* `data/jobs/<id>/score/local_gate.json`

Tier 0 responsibilities:

1. Normalize raw posting text
2. Compute `job_hash` using normalized text
3. Perform exact hash duplicate detection
4. Perform FAISS similarity duplicate detection using existing logic and threshold `0.82`
5. Run rule-based blockers:

   * location
   * visa
   * salary floor
   * contract type
   * title/domain allowlist and denylist
6. Run lightweight local scoring:

   * must-have skills match rate
   * years-of-experience phrase match
   * seniority mismatch penalty

Tier 0 decision values:

* `reject_early`
* `needs_llm`
* `strong_match` 

If `reject_early`, pipeline stops with **0 paid calls**. 

## Tier 1 — Single Consolidated LLM Call

Create `scripts/01_analyze_job.py`.

This replaces the separate score + extract phases and must do one consolidated paid call that returns:

* fit score
* recommendation
* rationale
* structured extraction
* brief resume tailoring plan

Required outputs:

* `data/jobs/<id>/score/llm_gate_report.md`
* `data/jobs/<id>/tailored/job_packet.json`
* metadata updates in `data/jobs/<id>/metadata.yaml` 

Tier 1 decision values:

* `skip`
* `hold`
* `proceed` 

Most jobs should stop here. 

## Tier 2 — Premium Single Package Call

Create `scripts/04_generate_application_package.py`.

Run Tier 2 only when all are true:

* Tier 1 decision = `proceed`
* score >= `78`
* job priority = `high` 

Tier 2 produces in one paid call:

* `generated/resume_intermediate_v1.json`
* `generated/cover_intermediate_v1.json` when requested
* `research/company_profile_v1.yaml`
* `generated/application_package_preview.md` 

No standalone company-research-only paid call in the main path.

## REQUIRED STATE MODEL

Do not overload one generic `decision` field. Use separate state fields in `metadata.yaml`:

* `job_hash`
* `profile_hash`
* `analysis_hash`
* `prompt_version`
* `model`
* `tier0_decision` = `reject_early|needs_llm|strong_match`
* `llm_decision` = `skip|hold|proceed`
* `user_decision` = `pending|accept|reject|hold`
* `score`
* `job_priority`
* `tier2_eligible`
* `artifact_schema_version`
* `generated_from_analysis_hash`

This separation is required so local gate, LLM gate, and human override do not conflict.

---

## REQUIRED CACHE / IDEMPOTENCY RULES

Before any paid call, check artifacts and hashes.

Cache key:

* `job_hash + profile_hash + prompt_version + model` 

Rules:

* If `tailored/job_packet.json` exists and cache fields match, skip Tier 1 paid call
* If `generated/resume_intermediate_v1.json` exists and cache fields match, skip Tier 2 resume generation
* If `generated/cover_intermediate_v1.json` exists and cache fields match, skip Tier 2 cover generation
* If only resume is requested, do not generate cover artifacts
* If cover is requested later and resume cache is valid, only generate missing cover path if architecture allows; otherwise rerun the single Tier 2 package only when necessary and record why

Every generated artifact must embed or sit next to enough provenance to prove which `analysis_hash` created it.
